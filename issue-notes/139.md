# issue chorusのargsが「配列、要素数3」しか受け取れないので、公式のfeedbackが指定できない #139
[issues #139](https://github.com/cat2151/tonejs-json-sequencer/issues/139)

# 考察

- そもそも公式リファレンスがわかりづらく、本当にfeedbackが指定できるのか？それとも実は引数は3つしか指定できないのか？がわかりづらい
    - 「配列、要素数3」の例しか書いていないため、どのように引数を指定すればよいか？のリファレンスが公式に存在しない
        - これではミスした場合の解決のヒントがないため、userの作業負荷が増大する

- 公式リファレンス
    - https://tonejs.github.io/docs/15.0.4/interfaces/ChorusOptions.html

# どうする？
- とにかく公式に書いてあるとおり、feedbackを含むargsを指定できるよう実装を試すこと
- ほかのinstrumentやeffectで、argsが配列でなく `{～}` で指定できたケースがある。それを参考にすること
- demoに追加で、chorus args not array みたいなやつを追加して、実際に動作するかを検証できるようにすること

# 備考

- 既存のissueでは、feedback等も指定して実装させたはずである。
- それをagentが無視した可能性がある。しかもそれをPRに報告せずにサイレントに実施してしまった可能性がある。
- 再発防止策のため、過去のissueとPRを調査せよ。
- これに関係するものを探せ。
- 見つからない場合は見つからないという報告をせよ。
